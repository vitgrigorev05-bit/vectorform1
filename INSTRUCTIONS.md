# 🚀 Инструкция по запуску ВекторФорм

## 📋 Краткое описание проекта

**ВекторФорм** — полнофункциональный MVP-прототип платформы для:
1. **Маркетплейса 3D-моделей** — покупка и продажа моделей
2. **Автоматического расчёта печати** — мгновенный расчёт стоимости
3. **Распределённого производства** — сеть партнёров-печатников

## 🛠️ Требования

- Node.js 18+
- PostgreSQL или SQLite (можно использовать встроенный Prisma Data Proxy)
- npm или yarn

## ⚡ Быстрый запуск

### 1. Установка зависимостей
```bash
cd vectorforms
npm install
```

### 2. Настройка базы данных
Для простоты тестирования используем встроенный Prisma Data Proxy (требуется интернет):

```bash
# Применить схему базы данных
npm run db:push

# Заполнить демо-данными (опционально)
npm run db:seed
```

### 3. Запуск проекта
```bash
npm run dev
```

Откройте [http://localhost:3000](http://localhost:3000) в браузере.

## 🏗️ Архитектура проекта

### Структура каталогов:
```
vectorforms/
├── app/                    # Next.js App Router
│   ├── (auth)/            # Страницы аутентификации
│   ├── (dashboard)/       # Личные кабинеты
│   ├── (marketplace)/     # Маркетплейс
│   ├── (print)/           # Печать и расчёт
│   ├── admin/             # Админ-панель
│   ├── api/               # API endpoints
│   └── layout.tsx         # Корневой layout
├── components/            # React компоненты
│   ├── ui/               # Базовые UI компоненты
│   └── layout/           # Компоненты layout
├── lib/                   # Утилиты и конфигурации
├── prisma/               # Схема базы данных
├── public/               # Статические файлы
└── styles/               # Глобальные стили
```

### Технологический стек:
- **Frontend**: Next.js 14 (App Router) + React + TypeScript
- **Styling**: Tailwind CSS + shadcn/ui
- **Backend**: Next.js API Routes
- **Database**: PostgreSQL + Prisma ORM
- **Authentication**: Mock система (реальная в следующих версиях)
- **UI Components**: shadcn/ui с кастомизацией
- **Charts**: Recharts для аналитики
- **3D Visualization**: Three.js (заглушки для прототипа)

### 3. Личные кабинеты
- [x] **Клиент**: заказы, загрузки, избранное
- [x] **Автор**: управление моделями, статистика, роялти
- [x] **Партнёр**: входящие заказы, оборудование, доходы
- [x] **Админ**: аналитика, управление, мониторинг

### 4. Система расчёта
```javascript
// Пример формулы
Базовая цена = (Объём × Цена материала) + (Время печати × Стоимость часа)

Коэффициенты:
- materialCoefficient: цена за см³ материала
- timeCoefficient: стоимость часа печати
- complexityMultiplier: множитель сложности (1.0-2.0)
- supportMultiplier: множитель за поддержки (1.0-1.5)
- postProcessingFee: плата за постобработку
- platformFee: комиссия платформы (%)
- partnerMargin: наценка партнёра (%)
- shippingCost: стоимость доставки
```

## 🆘 Поиск и устранение неисправностей

### Ошибка: "Database connection failed"
```bash
# Использовать SQLite для локальной разработки
# Изменить DATABASE_URL в .env на:
DATABASE_URL="file:./dev.db"
```

### Ошибка: "Module not found"
```bash
# Переустановить зависимости
rm -rf node_modules package-lock.json
npm install
```

### Ошибка: "Prisma migration failed"
```bash
# Сбросить базу данных
npx prisma migrate reset
# Или применить схему напрямую
npx prisma db push
```

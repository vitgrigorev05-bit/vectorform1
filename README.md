# ВекторФорм - Платформа для 3D-печати и маркетплейса моделей

Полнофункциональный MVP-прототип платформы для автоматического расчёта стоимости 3D-печати, маркетплейса 3D-моделей и распределённого производства через сеть партнёров.

## 🚀 Быстрый старт

### Установка зависимостей

```bash
cd vectorforms
npm install
```

### Настройка базы данных

1. Убедитесь, что у вас установлен PostgreSQL
2. Обновите DATABASE_URL в файле `.env`
3. Запустите миграции Prisma:

```bash
npm run db:push
```

### Запуск проекта

```bash
npm run dev
```

Откройте [http://localhost:3000](http://localhost:3000) в браузере.

## 🎯 Основные функции

### 1. Маркетплейс 3D-моделей
- Просмотр каталога моделей с фильтрами
- Бесплатные и платные модели
- Поиск по категориям и тегам
- Рейтинги и отзывы
- Избранное и корзина

### 2. Автоматический расчёт печати
- Загрузка 3D-моделей (STL, OBJ, 3MF и др.)
- Анализ геометрии модели
- Мгновенный расчёт стоимости
- Выбор материалов, цветов и качества
- Детализация стоимости по статьям

### 3. Сеть производственных партнёров
- Автоматическое распределение заказов
- Учёт региона, оборудования и загрузки
- Контроль качества и сроков
- Доставка готовых изделий

### 4. Личные кабинеты
- **Клиенты**: заказы, загрузки, избранное
- **Авторы**: загрузка моделей, статистика, роялти
- **Партнёры**: входящие заказы, календарь, доходы
- **Администраторы**: управление контентом, аналитика

## 🛠 Технологический стек

- **Frontend**: Next.js 14 (App Router) + React + TypeScript
- **Styling**: Tailwind CSS + shadcn/ui
- **Backend**: Next.js API Routes
- **Database**: PostgreSQL + Prisma ORM
- **Authentication**: NextAuth.js
- **3D Visualization**: Three.js + @react-three/fiber
- **Charts**: Recharts
- **Form Validation**: React Hook Form + Zod
- **State Management**: Zustand
- **File Upload**: UploadThing (заглушка)

## 📊 Сущности базы данных

Основные модели:
- `User` - пользователи (клиенты, авторы, партнёры, админы)
- `Model3D` - 3D-модели в маркетплейсе
- `ModelFile` - файлы моделей
- `Material` - материалы для печати
- `PrintOrder` - заказы на печать
- `Partner` - партнёры-печатники
- `Transaction` - транзакции и платежи
- `Review` - отзывы и рейтинги

## 🔧 Логика расчёта стоимости

Формула расчёта:
```
Базовая цена = (Объём × Цена материала) + (Время печати × Стоимость часа)

Коэффициенты:
- materialCoefficient: цена за см³ материала
- timeCoefficient: стоимость часа печати
- complexityMultiplier: множитель сложности (1.0-2.0)
- supportMultiplier: множитель за поддержки (1.0-1.5)
- postProcessingFee: плата за постобработку
- platformFee: комиссия платформы (%)
- partnerMargin: наценка партнёра (%)
- shippingCost: стоимость доставки
```

## 🔄 Логика распределения заказов

Алгоритм приоритетов:
1. Регион доставки (ближайший партнёр)
2. Доступность материала
3. Текущая загрузка (менее загруженные)
4. Рейтинг партнёра
5. Стоимость изготовления

## 🚢 API endpoints

- `GET /api/models` - список моделей маркетплейса
- `POST /api/models/upload` - загрузка модели
- `POST /api/print/calculate` - расчёт стоимости печати
- `POST /api/orders` - создание заказа
- `GET /api/partners` - список партнёров
- `POST /api/partners/assign` - назначение заказа партнёру
- `GET /api/dashboard/*` - данные кабинетов
